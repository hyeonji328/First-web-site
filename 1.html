태그

  1. html 태그는 최상위 태그. Html 파일의 모든 태그가 이 안에 들어감.
  2. head 태그는 본문을 설명하는 정보를 묶는 태그. title이나 meta 태그가 이 안에 들어감. body 태그는 본문을 묶는 태그. Html 파일의 모든 코드는 head나 body 아래 속해야 함.
  3. strong 태그는 글씨를 굵게 강조할 때 사용함.
  4. u 태그는 밑줄을 그을 때 사용함.
  5. h1부터 h6까지는 headings로 제목을 나타내고, 숫자가 작을수록 중요한 정보이며 웹에서 큰 글씨로 표현됨. h 태그를 쓰면 일반 글씨보다 굵게 표현됨.
  6. br 태그는 줄바꿈 태그. 단순한 줄바꿈이며 여러 번 써서 원하는 만큼 공백을 만들 수 있음. 닫는 태그 없이 여는 태그만 사용함.
  7. p 태그는 단락 태그. 여는 태그, 닫는 태그를 한 쌍으로 써야 함. br 태그보다 완성도 있는 태그. br 태그처럼 줄을 바꿀 수 있지만, 여러 번 사용할 수 없고, 정해진 대로 한 줄만 줄바꿈을 하기 때문에 시각적인 자유도가 떨어짐. 이걸 더 정교하게 보완할 때 쓸 수 있는 기술이 css; 위에서 사용한 style.
  8. img 태그는 이미지를 삽입하는 태그. 하지만 태그의 이름만으로는 정보가 부족해서 '속성'을 사용해야 함, 이 경우에 사용하는 속성은 src. 저작권 구애받지 않고 이미지 사용할 수 있는 유용한 사이트: unsplash.com.
  9. parent와 child. 포함하는 태그는 부모, 포함되는 태그는 자식. 둘의 관계는 바뀔 수도 있음.
  10. li 태그는 목차 list를 만드는 태그. 항상 부모 태그인 ul 태그나 ol 태그가 있음. ul 태그는 항상 li 태그의 부모 태그. 리스트의 어디서부터 어디까지가 하나인지 그룹핑하는 역할. unordered list. ol 태그는 자동으로 목차를 넘버링하는 태그. ordered list.
  11. table 태그는 표를 만드는 태그. 한 표에 한 쌍만 존재함. tr/table row/ 태그, th/table header/ 태그, td/table cell/ 태그와 함께 다니는 부모 태그. 자식 태그는 여러 개 가능.
  12. title 태그는 웹 페이지의 제목을 나타내는 태그. 브라우저가 책 제목의 표지와 같이 인식하는 것.
  13. meta 태그는 브라우저가 html 문서의 문자 인코딩 방식을 명시하는 태그. utf-8로 쓰여졌다면 meta 태그에 속성 charset을 써서 charset="utf-8"이라고 적으면 됨.
  14. a(anchor) 태그는 링크(Hyper Text)를 거는 태그.
  15. div 태그는 division의 약자로 레이아웃을 만들 때 사용함. 여러 속성을 추가해서 활용. 함께 쓰는 속성으로는 style, margin, width, height, background-color, float, border 등이 있음.
  16. input 태그는 입력 형식을 구현하기 위한 태그. 함께 쓰는 type 속성의 속성값에 따라 화면에 다양한 걸 표시할 수 있음.
  17. iframe 태그는 해당 웹페이지에 다른 html 파일을 불러와서 삽입, 합성할 수 있는 태그. src 속성으로 URL을 추가하고 height와 width 속성으로 크기 조절.
  18. !-- 콘텐츠 -- 를 쓰면 그 안에 있는 콘텐츠는 없는 셈치라고 명령하는 것.
  19. style 태그는 HTML의 문법이면서 동시에 태그 안의 내용을 CSS의 문법으로 해석하라고 컴퓨터에 명령함.


속성

태그만으로 부족한 정보를 채우는 게 속성. 여러 개를 쓸 수 있고 순서에 상관없이 결과값은 같음.

  1. style 속성은 태그의 스타일을 지정함. 이걸로 다른 속성을 쓸 수도 있음.
  2. src 속성은 태그의 속성 즉, URL을 의미함. 태그만으로는 어떤 정보를 넣을지 알 수 없음. 이때 부족한 정보를 추가하는 속성.
  3. width 속성은 가로 크기를 조절할 때 쓰는 속성.
  4. height 속성은 세로 크기를 조절할 때 쓰는 속성.
  5. charset은 meta 태그와 함께 쓰는 속성. 어떤 문자로 인코딩할지 명시.
  6. href(hyper text reference) 속성은 a 태그로 걸 링크의 정보를 제공하는 속성.
  7. target 속성은 a 태그로 건 링크가 열릴 위치를 명시하는 속성. _blank는 새 탭에서 열기, _self는 현재 프레임에서 열기로 기본값, _parent는 현재 프레임의 부모 프레임에서 열기, _top은 현재 윈도우 전체에서 열기, 프레임 이름을 적으면 그 프레임에서 열기.
  8. title 속성은 a 태그로 건 링크가 어떤 내용인지 툴팁으로 미리 알려 주는 속성.
  9. float 속성은 정렬을 좌우하는 속성, 가운데 정렬은 안 됨.
  10. margin 속성은 여백을 주는 속성. 위, 아래, 옆에 px이나 %로 수치를 입력해 여백을 줄 수 있음.
  11. border 속성은 테두리의 굵기를 정하는 속성. 숫자가 클수록 굵어짐.
  12. background-color 속성은 배경의 색을 정하는 속성.
  13. color는 글씨의 색을 바꾸는 속성. style 속성 안에 사용함.
  14. type 속성은 input 태그의 속성으로, 해당 태그가 나타낼 타입을 명시함. 다양한 속성값이 있음. checkbox는 체크박스를 정의함. 속성값은 button, color, date, datetime-local, email, file, hidden, image, month, number, password, radio, range, reset, search, submit, tel, text, time, url, week 등.
  15. class 속성은 어떠한 요소들을 하나의 클래스로 묶어 동일한 스타일을 적용하기 위해서 사용함. 속성값 saw를 사용하면 보았던 요소라는 정보를 줄 수 있음. 속성값 active는 지금 실행하고 있는 페이지라는 뜻. class에는 여러 개의 속성값을 쓸 수 있지만, 최근에 명령한 것(제일 마지막에 적은 것)에 우선순위가 적용된다는 단점이 있음.
  16. id 속성은 속성값에 절대적인 우위를 주는 속성. 따라서 id 속성에 쓴 속성값은 동일 문서 내에서 다시 쓰면 안 됨. id의 속성값은 class의 속성값보다 나중에 등장하더라도 순위가 밀리지 않고 적용됨.


웹페이지 만들기

  1. 웹사이트를 링크로 엮으면 웹페이지가 됨. 목차에 링크를 달고, 링크 정보 속성에 각 목차에 맞는 html 문서(웹사이트) 링크를 넣어 주면 됨.
  2. 이 파일이 html 파일이라는 의미에서 제일 위에 관용적으로 '!doctype html'이라고 써 주면 됨.
  3. 댓글 기능 구현하기: DISQUS, LiveRe 등의 서비스를 사용해 웹페이지에 댓글 기능을 구현할 수 있음. 댓글 기능은 백엔드 기술이 필요하지만 이러한 서비스로 한번 구현해 볼 수는 있는 것. 다만 DISQUS를 이용하면 파일 형태로 열었을 땐 보안상의 이유로 서비스를 사용할 수 없음. 웹 서버로 열어야 작동함.
  4. 채팅 서비스 구현하기: tawk라는 서비스로 구현 가능.웹 서버로 열어야 작동함.
  5. 웹페이지 사용자 분석하기: Google Analytics를 활용해 사용자를 분석할 수 있음. 사용자 메뉴는 연령, 성별, 국가, 장치 등 사용자의 정보를 보여줌. 수명 주기는 사용자가 사이트에서 한 일을 보여주는 정보를 제공함. 인구 통계와 수명 주기에서 중요한 건 시간.


웹페이지 배포

  1. 웹 호스팅

    - 통신 업체나 전문 웹 개발 회사에서 서버를 임대해 주는 것. 인터넷에 연결된 각 컴퓨터(host)에 소프트웨어까지 빌려주는 사업. Cloud라고도 부름. 쉽지만 배울 수 있는 게 적음.

    - GitHub: 개발자들이 안전하게 소스코드를 보관하고 협업하려고 만들었던 사이트.

  2. 웹 서버

    - 자신의 컴퓨터에 웹 서버를 직접 설치하는 것. 어렵지만 배울 수 있는 게 많음.

    - Live Server: 웹 서버의 일종. VS code의 확장 기능. 변경 사항이 실시간으로 적용된다는 특징이 있음. Go live 버튼으로 키고, Port 버튼으로 끔.

  3. 웹 호스팅으로 시작해서 웹 서버를 통해 원리를 공부하기.
  4. 웹 브라우저가 문서를 직접 열면 주소가 file:///Desktop/web/index.html로 뜸. 웹 서버를 통해 문서를 열면 주소는 http://127.0.0.1.5000/index.html로 뜸.
  5. http://127.0.0.1.5000/index.html
    • http: Hypertext Transfer Protocol, 하이퍼텍스트 통신 규약.
    • 127.0.0.1: Internet Protocol Address(인터넷에서 사용하는 주소, ip)의 일종. 약 43억 개가 있고, 그중 127.0.0.1은 전 세계에서 자신의 컴퓨터를 나타낼 때 쓰기로 약속한 ip. 따라서 주소창에 이 ip가 떠 있다면, 웹 브라우저가 자기 컴퓨터의 웹 서버에 접속하려고 시도하고 있다는 의미. 주로 개발할 때 많이 사용함.
    • Port: 하나의 컴퓨터는 여러 웹 서버와 연결되어 있음. 이때 각 서버는 각자의 포트 번호를 가지고 있음.
  6. 모바일 웹 브라우저에서 컴퓨터의 웹 서버에 접속하기

    - 같은 네트워크를 공유하고 있어야 함.

    - 컴퓨터의 ip 주소와 포트 번호를 모바일 웹 브라우저에 입력하면 접속됨.



개념

  1. Web Browser: 정보를 요청하는 client computer. 웹 클라이언트라고 불러도 OK.
  2. Web Server: 요청에 응답해 정보를 전송하는 server computer.
  3. Ex: 게임 클라이언트-게임 서버, 채팅 클라이언트-채팅 서버 등
